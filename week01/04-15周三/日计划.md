## 学习任务清单

### 任务1：数组与链表（20分钟）

**数组（顺序存储）**：

| 特性 | 说明 |
|------|------|
| 存储方式 | 连续内存空间 |
| 访问时间 | O(1) - 直接计算地址 |
| 插入删除 | O(n) - 需移动元素 |
| 适用场景 | 频繁访问、少修改 |

**数组地址计算**：
```
a[i]地址 = 基地址 + i × 元素大小
```

**链表（链式存储）**：

| 特性 | 说明 |
|------|------|
| 存储方式 | 分散内存，指针链接 |
| 访问时间 | O(n) - 需遍历 |
| 插入删除 | O(1) - 只改指针 |
| 适用场景 | 频繁插入删除 |

**链表类型**：
- 单链表：单向指针
- 双链表：双向指针
- 循环链表：首尾相连

---

### 任务2：栈（20分钟）⭐重点

**栈的特性**：
- **后进先出（LIFO）**
- 只能在一端（栈顶）操作
- 操作：push（入栈）、pop（出栈）

**栈的应用场景**：

| 应用 | 说明 |
|------|------|
| 函数调用 | 保存返回地址、参数、局部变量 |
| 递归实现 | 保存递归调用栈 |
| 表达式求值 | 后缀表达式计算 |
| 括号匹配 | 检查括号是否匹配 |
| 页面浏览历史 | 浏览器后退功能 |

**栈的考点**：
- 入栈顺序 → 出栈顺序判断
- n个元素进栈，合法出栈序列数量 = Catalan数 = C(2n,n)/(n+1)

---

### 任务3：队列（20分钟）⭐重点

**队列的特性**：
- **先进先出（FIFO）**
- 队头出队、队尾入队
- 操作：enqueue（入队）、dequeue（出队）

**队列的应用场景**：

| 应用 | 说明 |
|------|------|
| 任务调度 | CPU任务队列 |
| 打印队列 | 打印任务排队 |
| 消息队列 | 生产者-消费者模型 |
| 缓冲区 | 数据缓冲 |
| BFS遍历 | 图的广度优先搜索 |

**循环队列**：
- 解决普通队列"假溢出"问题
- 队满判断：(rear+1) % n == front
- 队空判断：rear == front

---

## 对比总结表

| 结构 | 存储方式 | 插入删除 | 访问 | 特性 | 典型应用 |
|------|----------|----------|------|------|----------|
| 数组 | 顺序 | O(n) | O(1) | 固定大小 | 随机访问 |
| 链表 | 链式 | O(1) | O(n) | 动态大小 | 频繁增删 |
| 栈 | 顺序/链式 | O(1) | - | LIFO | 函数调用、递归 |
| 队列 | 顺序/链式 | O(1) | - | FIFO | 任务调度、BFS |

## 练习题目

1. 栈的入栈顺序是1,2,3,4,5，以下哪个出栈序列合法？
   - A. 5,4,3,2,1 ✅
   - B. 4,3,5,1,2 ❌（5必须在1、2之前）
   - C. 3,4,5,2,1 ✅

2. 循环队列大小为n，队满条件是？
   - 答案：(rear+1) % n == front

3. 以下哪个场景用栈更合适？
   - 函数调用返回地址保存 ✅
   - CPU任务调度 ❌（用队列）

---

## 易错点提醒

1. 栈判断出栈合法性：不能"先出后进"
2. 循环队列牺牲一个单元判断队满
3. BFS用队列，DFS用栈
