## 任务1：图的基本概念（60分钟）

**图的定义**：
- 图G = (V, E)
- V：顶点集合
- E：边集合

**图的分类**：

| 类型 | 特点 | 示例 |
|------|------|------|
| 无向图 | 边无方向 | 道路网络 |
| 有向图 | 边有方向 | 网页链接 |
| 无权图 | 边无权重 | 简单关系 |
| 有权图 | 边有权重 | 路程距离 |

**图示对比**：
```
无向图：              有向图：              有权图：
    A --- B              A ──→ B              A --5-- B
    |     |              |     ↓              |       |
    C --- D              C ←── D              C --3-- D

说明：               说明：               说明：
- 边没有箭头         - 边有箭头方向        - 边上有数字表示权重
- A-B 等价于 B-A     - A→B 不等于 B→A     - 表示距离、代价等
```

**图的基本术语**：

| 术语 | 定义 |
|------|------|
| 度 | 顶点关联的边数 |
| 入度 | 有向图中进入该顶点的边数 |
| 出度 | 有向图中从该顶点出发的边数 |
| 路径 | 从一个顶点到另一个顶点的边序列 |
| 连通图 | 任意两顶点都有路径相连 |
| 强连通图 | 有向图中任意两顶点双向连通 |
| 完全图 | 每两个顶点之间都有边 |

**术语图解**：

**度、入度、出度**：
```
无向图（度）：         有向图（入度/出度）：
    A                   A ← B
   / \                  ↑   ↑
  B   C                 |   |
   \ /                  D → C
    D

A的度 = 2（2条边）      A：入度=1（D→A），出度=0
B的度 = 2（2条边）      B：入度=0，出度=1（B→A）
C的度 = 2（2条边）      C：入度=2（D→C, B→A? 不对）
D的度 = 2（2条边）      D：入度=0，出度=2（D→A, D→C）

等等，让我重新画一个更清晰的：
```

**正确的有向图示例**：
```
    A ←── B
    ↑     ↑
    |     |
    D ──→ C

边：B→A, D→A, D→C, C→B

入度（箭头指向自己）：
- A：入度=2（B→A, D→A）
- B：入度=1（C→B）
- C：入度=1（D→C）
- D：入度=0

出度（箭头从自己出发）：
- A：出度=0
- B：出度=1（B→A）
- C：出度=1（C→B）
- D：出度=2（D→A, D→C）

验证：总入度 = 总出度 = 边数 = 4 ✓
```

**连通图 vs 非连通图**：
```
连通图：               非连通图：
    A --- B              A --- B      D
    |     |              |            |
    C --- D              C            E

任意两点都有路径        D、E与A、B、C无路径
```

**强连通图**（有向图特有）：
```
强连通图：              非强连通图：
    A ⇄ B                  A → B
    ⇄   ⇄                  ↑   ↓
    D ⇄ C                  D ← C

强连通：每对顶点双向都有路径
- A→B：直接有边
- B→A：直接有边
- A→C：A→B→C 或 A→D→C
- C→A：C→B→A 或 C→D→A
- 任意两点双向可达 ✓

非强连通：
- A→D：无路径（A→B→C，但C→D没有边）
- D→A：有路径（D→C→B→A）
- A无法到达D，所以不是强连通 ✗
```

**强连通的关键**：任意两点之间，**双向**都有路径，不一定要有直接的边。

**完全图的边数**：
- 无向完全图：n(n-1)/2 条边
- 有向完全图：n(n-1) 条边

**完全图图解**：

**无向完全图（4个顶点）**：
```
        A
       /|\      ← A连接B、C、D（3条边）
      / | \
     /  |  \
    B---+---D   ← B连接A、C、D
     \  |  /
      \ | /
       \|/      ← C连接A、B、D
        C

列出所有边：
  A-B  A-C  A-D  （从A出发）
  B-C  B-D       （从B出发，A-B已算过）
  C-D            （从C出发，前面已算过）

总共 6 条边 ✓
公式：n(n-1)/2 = 4×3/2 = 6
```

**有向完全图（3个顶点）**：
```
从每个顶点出发的边（单独看）：

A出发：                B出发：                C出发：
  A → B                B → A                C → A
  A → C                B → C                C → B
 (2条边)               (2条边)               (2条边)

合在一起看：
      A
      ↓ ↑
    ↙   ↘
   B ←─→ C
    ↖   ↗

每个顶点都有2条出边（指向其他两个顶点）
3个顶点 × 2条出边 = 6条边 ✓

公式：n(n-1) = 3×2 = 6
```

**理解方法**：
```
有向完全图 = 每个顶点都要"指向"所有其他顶点

3个顶点的情况：
- A要指向B和C → 2条边
- B要指向A和C → 2条边
- C要指向A和B → 2条边

总共：3 × 2 = 6条边
```

**列出所有边**：
| 边 | 说明 |
|----|----|
| A→B | A指向B |
| A→C | A指向C |
| B→A | B指向A |
| B→C | B指向C |
| C→A | C指向A |
| C→B | C指向B |

总共 6 条边 ✓
公式：n(n-1) = 3×2 = 6
```

**核心区别**：

| 类型 | 每对顶点之间 | 公式 | 理解 |
|------|-------------|------|------|
| 无向完全图 | **1条边**（共享） | n(n-1)/2 | 一条线连两端，除以2 |
| 有向完全图 | **2条边**（双向） | n(n-1) | 两条箭头各算各，不除 |

**最简单例子对比**：
```
无向图（2顶点）：          有向图（2顶点）：
    A ─── B                A ⇄ B
    1条边                  2条边（A→B + B→A）

公式：2×1/2 = 1 ✓         公式：2×1 = 2 ✓
```

## 任务2：图的存储方式（30分钟）⭐重点

**邻接矩阵**：

| 特性 | 说明 |
|------|------|
| 存储方式 | 二维数组 |
| 空间复杂度 | O(n²) |
| 适用场景 | 稠密图、频繁查询边 |
| 判断边存在 | O(1) |
| 查找所有邻接点 | O(n) |

**示例**：
```
无向图：
    A --- B
    |   /
    |  /
    C

边：A-B, A-C, B-C

邻接矩阵：
      A  B  C
  A [ 0  1  1 ]
  B [ 1  0  1 ]
  C [ 1  1  0 ]

解读：
- matrix[i][j] = 1 表示有边
- matrix[i][j] = 0 表示无边
- 无向图矩阵对称（A-B 和 B-A 是同一条边）
```

**邻接表**：

| 特性 | 说明 |
|------|------|
| 存储方式 | 链表数组 |
| 空间复杂度 | O(n+e) |
| 适用场景 | 稀疏图、频繁遍历 |
| 判断边存在 | O(d)（d为度） |
| 查找所有邻接点 | O(d) |

**⭐ 邻接表怎么写？（详细步骤）**

**核心思想**：为每个顶点建一个"邻居名单"

```
邻接表 = 数组 + 链表

数组：存放所有顶点
链表：每个顶点后面挂一个链表，存它的邻居
```

**写法步骤**：

```
Step1：列出所有顶点作为"头"
       A  B  C  D

Step2：看每个顶点有哪些邻居，写在后面

       A的邻居是谁？看图：
       - A连着B → 写B
       - A连着C → 写C
       所以：A → B → C → ∅

       B的邻居是谁？
       - B连着A → 写A
       - B连着D → 写D
       所以：B → A → D → ∅

       C的邻居是谁？
       - C连着A → 写A
       - C连着D → 写D
       所以：C → A → D → ∅

       D的邻居是谁？
       - D连着B → 写B
       - D连着C → 写C
       所以：D → B → C → ∅
```

**图解对照**：
```
原图：
    A --- B
    |     |
    C --- D

边：A-B, A-C, B-D, C-D

邻接表写法：
顶点数组：[A, B, C, D]

     A → [B] → [C] → ∅     （A有两个邻居：B、C）
     B → [A] → [D] → ∅     （B有两个邻居：A、D）
     C → [A] → [D] → ∅     （C有两个邻居：A、D）
     D → [B] → [C] → ∅     （D有两个邻居：B、C）

空格→表示链表指针，∅表示链表结束（空指针）
```

**有向图的邻接表写法**：
```
有向图：
    A → B
    ↓
    C → D

边：A→B, A→C, C→D

邻接表（只写"出边"指向的邻居）：
     A → [B] → [C] → ∅    （A指向B和C）
     B → ∅                （B没有出边）
     C → [D] → ∅          （C指向D）
     D → ∅                （D没有出边）

注意：有向图只写"箭头指向"的邻居，不写"箭头指向自己"的
```

**邻接矩阵 vs 邻接表 对比**：

```
同一个图：
    A --- B
    |     |
    C --- D

邻接矩阵（4×4数组）：          邻接表（4个链表）：
      A  B  C  D                A → B → C → ∅
  A [ 0  1  1  0 ]              B → A → D → ∅
  B [ 1  0  0  1 ]              C → A → D → ∅
  C [ 1  0  0  1 ]              D → B → C → ∅
  D [ 0  1  1  0 ]

空间：16个单元                  空间：4+8=12个单元
（固定n²）                      （顶点+边数×2）
```

**选择建议**：

| 情况 | 选择 | 原因 |
|------|------|------|
| 稠密图（边多） | 邻接矩阵 | 空间利用率高 |
| 稀疏图（边少） | 邻接表 | 节省空间 |
| 频繁查询边 | 邻接矩阵 | O(1)直接访问 |
| 频繁遍历邻接点 | 邻接表 | O(d)只访问存在的边 |

## 任务3：练习题目（30分钟）

1. 5个顶点的无向完全图有多少条边？
   - 答案：n(n-1)/2 = 5×4/2 = 10

2. 有向图中，所有顶点入度之和等于？
   - 答案：等于出度之和，等于边数

3. 稀疏图用什么存储更节省空间？
   - 答案：邻接表

4. 需要频繁判断两顶点是否有边，用什么存储？
   - 答案：邻接矩阵
