## 任务1：图的基本概念（60分钟）

**图的定义**：
- 图G = (V, E)
- V：顶点集合
- E：边集合

**图的分类**：

| 类型 | 特点 | 示例 |
|------|------|------|
| 无向图 | 边无方向 | 道路网络 |
| 有向图 | 边有方向 | 网页链接 |
| 无权图 | 边无权重 | 简单关系 |
| 有权图 | 边有权重 | 路程距离 |

**图示对比**：
```
无向图：              有向图：              有权图：
    A --- B              A ──→ B              A --5-- B
    |     |              |     ↓              |       |
    C --- D              C ←── D              C --3-- D

说明：               说明：               说明：
- 边没有箭头         - 边有箭头方向        - 边上有数字表示权重
- A-B 等价于 B-A     - A→B 不等于 B→A     - 表示距离、代价等
```

**图的基本术语**：

| 术语 | 定义 |
|------|------|
| 度 | 顶点关联的边数 |
| 入度 | 有向图中进入该顶点的边数 |
| 出度 | 有向图中从该顶点出发的边数 |
| 路径 | 从一个顶点到另一个顶点的边序列 |
| 连通图 | 任意两顶点都有路径相连 |
| 强连通图 | 有向图中任意两顶点双向连通 |
| 完全图 | 每两个顶点之间都有边 |

**术语图解**：

**度、入度、出度**：
```
无向图（度）：         有向图（入度/出度）：
    A                   A ← B
   / \                  ↑   ↑
  B   C                 |   |
   \ /                  D → C
    D

A的度 = 2（2条边）      A：入度=1（D→A），出度=0
B的度 = 2（2条边）      B：入度=0，出度=1（B→A）
C的度 = 2（2条边）      C：入度=2（D→C, B→A? 不对）
D的度 = 2（2条边）      D：入度=0，出度=2（D→A, D→C）

等等，让我重新画一个更清晰的：
```

**正确的有向图示例**：
```
    A ←── B
    ↑     ↑
    |     |
    D ──→ C

边：B→A, D→A, D→C, C→B

入度（箭头指向自己）：
- A：入度=2（B→A, D→A）
- B：入度=1（C→B）
- C：入度=1（D→C）
- D：入度=0

出度（箭头从自己出发）：
- A：出度=0
- B：出度=1（B→A）
- C：出度=1（C→B）
- D：出度=2（D→A, D→C）

验证：总入度 = 总出度 = 边数 = 4 ✓
```

**连通图 vs 非连通图**：
```
连通图：               非连通图：
    A --- B              A --- B      D
    |     |              |            |
    C --- D              C            E

任意两点都有路径        D、E与A、B、C无路径
```

**强连通图**（有向图特有）：
```
强连通图：              非强连通图：
    A ⇄ B                  A → B
    ⇄   ⇄                  ↑   ↓
    D ⇄ C                  D ← C

强连通：每对顶点双向都有路径
- A→B：直接有边
- B→A：直接有边
- A→C：A→B→C 或 A→D→C
- C→A：C→B→A 或 C→D→A
- 任意两点双向可达 ✓

非强连通：
- A→D：无路径（A→B→C，但C→D没有边）
- D→A：有路径（D→C→B→A）
- A无法到达D，所以不是强连通 ✗
```

**强连通的关键**：任意两点之间，**双向**都有路径，不一定要有直接的边。

**完全图的边数**：
- 无向完全图：n(n-1)/2 条边
- 有向完全图：n(n-1) 条边

**完全图图解**：
```
无向完全图（4个顶点）：      有向完全图（3个顶点）：
    A ─── B                    A ⇄ B
    │╲   ╱│                    ↕ ╲ ╱ ↕
    │ ╲ ╱ │                    ╲ ↕ ╱
    C ─── D                     C

边数 = 4×3/2 = 6条          边数 = 3×2 = 6条

每对顶点之间都有1条边      每对顶点之间都有2条边（双向）
```

## 任务2：图的存储方式（30分钟）⭐重点

**邻接矩阵**：

| 特性 | 说明 |
|------|------|
| 存储方式 | 二维数组 |
| 空间复杂度 | O(n²) |
| 适用场景 | 稠密图、频繁查询边 |
| 判断边存在 | O(1) |
| 查找所有邻接点 | O(n) |

**示例**：
```
无向图：
    A --- B
    |   /
    |  /
    C

边：A-B, A-C, B-C

邻接矩阵：
      A  B  C
  A [ 0  1  1 ]
  B [ 1  0  1 ]
  C [ 1  1  0 ]

解读：
- matrix[i][j] = 1 表示有边
- matrix[i][j] = 0 表示无边
- 无向图矩阵对称（A-B 和 B-A 是同一条边）
```

**邻接表**：

| 特性 | 说明 |
|------|------|
| 存储方式 | 链表数组 |
| 空间复杂度 | O(n+e) |
| 适用场景 | 稀疏图、频繁遍历 |
| 判断边存在 | O(d)（d为度） |
| 查找所有邻接点 | O(d) |

**邻接矩阵 vs 邻接表 对比**：

```
同一个图：
    A --- B
    |     |
    C --- D

邻接矩阵（4×4数组）：          邻接表（4个链表）：
      A  B  C  D                A → B → C → ∅
  A [ 0  1  1  0 ]              B → A → D → ∅
  B [ 1  0  0  1 ]              C → A → D → ∅
  C [ 1  0  0  1 ]              D → B → C → ∅
  D [ 0  1  1  0 ]

空间：16个单元                  空间：4+8=12个单元
（固定n²）                      （顶点+边数×2）
```

**选择建议**：

| 情况 | 选择 | 原因 |
|------|------|------|
| 稠密图（边多） | 邻接矩阵 | 空间利用率高 |
| 稀疏图（边少） | 邻接表 | 节省空间 |
| 频繁查询边 | 邻接矩阵 | O(1)直接访问 |
| 频繁遍历邻接点 | 邻接表 | O(d)只访问存在的边 |

## 任务3：练习题目（30分钟）

1. 5个顶点的无向完全图有多少条边？
   - 答案：n(n-1)/2 = 5×4/2 = 10

2. 有向图中，所有顶点入度之和等于？
   - 答案：等于出度之和，等于边数

3. 稀疏图用什么存储更节省空间？
   - 答案：邻接表

4. 需要频繁判断两顶点是否有边，用什么存储？
   - 答案：邻接矩阵
