# 数制转换笔记

## 一、十进制转二进制

### 整数部分：除 2 取余法

**步骤**：
1. 十进制数 ÷ 2
2. 记录**余数**（0 或 1）
3. 用**商**继续 ÷ 2
4. 重复，直到商为 0
5. 余数**从下往上**读取

**示例：11₁₀ → ( ? )₂**

```
11 ÷ 2 = 5 ······ 1  ← 最低位（LSB）
 5 ÷ 2 = 2 ······ 1
 2 ÷ 2 = 1 ······ 0
 1 ÷ 2 = 0 ······ 1  ← 最高位（MSB）
─────────────────
结果：11₁₀ = 1011₂
```

---

### 小数部分：乘 2 取整法

**步骤**：
1. 小数 × 2
2. 记录**整数部分**（0 或 1）
3. 用**小数部分**继续 × 2
4. 重复，直到小数为 0 或达到精度
5. 整数**从上往下**读取

**示例：0.25₁₀ → ( ? )₂**

```
0.25 × 2 = 0.50  → 整数 = 0
0.50 × 2 = 1.00  → 整数 = 1  ← 小数为 0，结束
─────────────────
结果：0.25₁₀ = 0.01₂
```

**示例：0.625₁₀ → ( ? )₂**

```
0.625 × 2 = 1.25  → 1
0.25  × 2 = 0.50  → 0
0.50  × 2 = 1.00  → 1  ← 结束
─────────────────
结果：0.625₁₀ = 0.101₂

验证：0.101₂ = 1/2 + 0/4 + 1/8 = 0.5 + 0.125 = 0.625 ✓
```

## 二、二进制转八进制

### 三位分组法

**原理**：8 = 2³，每 3 位二进制对应 1 位八进制

**对应表**：

| 二进制 | 八进制 |
|--------|--------|
| 000    | 0      |
| 001    | 1      |
| 010    | 2      |
| 011    | 3      |
| 100    | 4      |
| 101    | 5      |
| 110    | 6      |
| 111    | 7      |

**示例：(101110)₂ → ( ? )₈**

```
第 1 步：从右往左三位分组
101110 → 101 | 110

第 2 步：每组转八进制
101₂ = 5₈
110₂ = 6₈

第 3 步：拼起来
结果：(101110)₂ = (56)₈
```

**注意**：最左边不足 3 位时补 0

```
(1011)₂ → 001 011 → 1 3 → (13)₈
```

## 三、二进制转十六进制

### 四位分组法

**原理**：16 = 2⁴，每 4 位二进制对应 1 位十六进制

**对应表**：

| 二进制 | 十六进制 | 二进制 | 十六进制 |
|--------|----------|--------|----------|
| 0000   | 0        | 1000   | 8        |
| 0001   | 1        | 1001   | 9        |
| 0010   | 2        | 1010   | A        |
| 0011   | 3        | 1011   | B        |
| 0100   | 4        | 1100   | C        |
| 0101   | 5        | 1101   | D        |
| 0110   | 6        | 1110   | E        |
| 0111   | 7        | 1111   | F        |

**示例：(10111011)₂ → ( ? )₁₆**

```
10111011 → 1011 | 1011
           B   |   B
结果：(BB)₁₆
```

## 四、浮点数表示

### 公式

```
N = M × R^E

N：浮点数的值
M：尾数（Mantissa）
R：基数（Radix）- 二进制中 R=2
E：阶码/指数（Exponent）
```

### 类比科学计数法

```
十进制：123.45 = 1.2345 × 10²
二进制：1011.01 = 1.01101 × 2³
```

---

### IEEE 754 浮点数结构（32 位单精度）

```
┌────────┬────────────────┬───────────────────────┐
│ 符号位 │     阶码       │        尾数          │
│ (1 位)  │    (8 位)      │      (23 位)         │
│   S    │      E         │         M            │
└────────┴────────────────┴───────────────────────┘
 31     30              23                     0
```

| 部分 | 作用 | 说明 |
|------|------|------|
| 符号位 S | 正负号 | 0=正数，1=负数 |
| 阶码 E | 指数部分 | 带偏移量存储（32 位偏移 127） |
| 尾数 M | 有效数字 | 只存小数部分（整数 1 隐含） |

---

### 浮点数转换示例：11.25₁₀

**第 1 步：转二进制**

```
11.25₁₀ = 1011.01₂

整数：11 = 1011₂
小数：0.25 = 0.01₂ （见小数转换）
```

**第 2 步：规格化**

```
1011.01 = 1.01101 × 2³
```

**第 3 步：计算各字段**

```
符号位 S = 0（正数）

阶码 E = 3 + 127 = 130 = 10000010₂

尾数 M = 01101... （去掉整数 1，存小数部分，补 0 到 23 位）
       = 01101000000000000000000
```

**第 4 步：拼接**

```
0 10000010 01101000000000000000000
↑ ────┬─── ──────────┬──────────
S    阶码 E           尾数 M
```

## 五、方法总结

| 转换类型 | 方法 | 读取方向 |
|----------|------|----------|
| 十进制整数→二进制 | 除 2 取余 | 从下往上 |
| 十进制小数→二进制 | 乘 2 取整 | 从上往下 |
| 二进制→八进制 | 三位分组 | 从左往右 |
| 二进制→十六进制 | 四位分组 | 从左往右 |

**口诀**：
- 整数除 2 取余，倒序
- 小数乘 2 取整，顺序
- 二转八，三位一组
- 二转十六，四位一组
