### 任务1：树的基本概念（15分钟）

**树的定义**：
- n个节点的有限集合
- 有且仅有一个根节点
- 其他节点分成若干互不相交的子树

**树的基本术语**：

| 术语 | 定义 | 示例 |
|------|------|------|
| 根节点 | 没有父节点的节点 | A |
| 叶子节点 | 没有子节点的节点 | D、E、F、G |
| 度 | 节点的子节点数量 | A的度为3 |
| 树的度 | 所有节点度的最大值 | 该树度为3 |
| 深度/高度 | 从根到节点的最长路径 | 树深度为3 |
| 层次 | 从根开始编号，根为第1层 | A在1层 |

**⭐ 度 vs 深度/高度 的区别**：

这是两个完全不同维度的概念：

| 概念 | 关注点 | 回答的问题 | 方向 |
|------|--------|-----------|------|
| **度** | 横向分支 | "一个节点最多有几个孩子？" | 宽度 |
| **深度/高度** | 纵向层次 | "树有多少层？" | 深度 |

示例图解：
```
        A           ← A的度=3，深度=0，高度=2
       /|\
      B C D         ← B的度=2, C的度=0, D的度=1，深度=1
     / \   \
    E   F   G       ← E,F,G的度=0，深度=2

树的度 = max(3,2,0,1,0,0,0) = 3  （看最宽的地方）
树的高度 = 3层                    （看最深的地方）
```

**记忆口诀**：度看"宽"，高度看"深"

### 任务2：二叉树性质（30分钟）⭐重点必考

**五大性质**：

**性质1**：第i层最多有 2^(i-1) 个节点
- 第1层：最多1个
- 第2层：最多2个
- 第3层：最多4个

**性质2**：深度为k的二叉树最多有 2^k - 1 个节点
- 深度1：最多1个
- 深度2：最多3个
- 深度3：最多7个

**⭐ "节点"是什么？**

节点 = 二叉树中的每个元素（圆圈）

```
        A           ← 1个节点（根节点）
       / \
      B   C         ← 2个节点
     / \
    D   E           ← 2个节点

总共5个节点：A, B, C, D, E
```

**性质1 vs 性质2 的区别**：

| 性质 | 问的问题 | 计算范围 |
|------|---------|---------|
| 性质1 | 某一层最多有多少节点？ | 单层 |
| 性质2 | 整棵树最多有多少节点？ | 全树 |

图解理解：
```
性质1（单层）：
第1层：    ●           → 最多 2^0 = 1 个节点
          / \
第2层：  ●   ●         → 最多 2^1 = 2 个节点
        /\  /\
第3层： ●● ●●          → 最多 2^2 = 4 个节点

性质2（全树）：
深度3的满二叉树：
        ●
       / \
      ●   ●           → 总共 2^3 - 1 = 7 个节点
     /\  /\
    ●● ●●
```

**性质3**：叶子节点数n0 = 度为2的节点数n2 + 1

**符号含义**：

| 符号 | 含义 | 度数 |
|------|------|------|
| n0 | 叶子节点的数量 | 度为0 |
| n1 | 只有1个孩子的节点数量 | 度为1 |
| n2 | 有2个孩子的节点数量 | 度为2 |

**核心公式**：`n0 = n2 + 1`（叶子节点永远比度为2的节点多1个）

```
总节点数 n = n0 + n1 + n2
```

**图解示例**：
```
        A           ← 度为2（有B、C两个孩子）
       / \
      B   C         ← B度为2，C度为0（叶子）
     / \
    D   E           ← D度为0，E度为0（都是叶子）

统计：
- n0 = 3（C、D、E是叶子，度为0的节点有3个）
- n1 = 0（没有度为1的节点）
- n2 = 2（A和B度为2，这样的节点有2个）

验证公式：n0 = n2 + 1 → 3 = 2 + 1 ✓
总节点数：n = n0 + n1 + n2 = 3 + 0 + 2 = 5 ✓
```

**易错点**：公式比较的是**节点个数**，不是度数本身！
- n0 = 度为0的节点**有多少个**
- n2 = 度为2的节点**有多少个**

**为什么成立？**（用分支数理解）
```
总分支数 = n1 + 2×n2    （度为1的贡献1条分支，度为2的贡献2条）
总分支数 = n - 1        （n个节点用n-1条分支连起来）

所以：n1 + 2×n2 = n - 1
    → n1 + 2×n2 = n0 + n1 + n2 - 1
    → n0 = n2 + 1
```

**考题常见用法**：
- 已知n2=30，求n0 → n0 = 31
- 已知n=100, n2=30，求n1 → n0=31, n1 = 100-31-30 = 39

**性质4**：n个节点的完全二叉树深度为 ⌊log₂(n)⌋ + 1

**符号解释**：

| 符号 | 含义 |
|------|------|
| n | 节点总数 |
| log₂(n) | 以2为底的对数 |
| ⌊x⌋ | 向下取整（地板函数） |

**计算示例**：

| n（节点数） | log₂(n) | ⌊log₂(n)⌋ | 深度 = ⌊log₂(n)⌋ + 1 |
|------------|---------|-----------|---------------------|
| 1 | 0 | 0 | 1 |
| 2 | 1 | 1 | 2 |
| 3 | 1.58 | 1 | 2 |
| 4 | 2 | 2 | 3 |
| 7 | 2.81 | 2 | 3 |
| 8 | 3 | 3 | 4 |
| 100 | 6.64 | 6 | 7 |

**图解理解**：
```
n=3时：        n=4时：         n=7时：
    ●              ●              ●
   / \            / \            / \
  ●   ●          ●   ●          ●   ●
                 /              / \ / \
                ●              ● ● ● ●
深度=2         深度=3          深度=3
```

**为什么需要向下取整？**

完全二叉树最后一层可能不满，所以节点数在某个范围内深度相同：
```
深度3的完全二叉树：n∈[4,7] 时深度都是3
⌊log₂(4)⌋ + 1 = 2 + 1 = 3
⌊log₂(7)⌋ + 1 = 2 + 1 = 3
```

**考题示例**：完全二叉树有100个节点，深度 = ⌊log₂(100)⌋ + 1 = 6 + 1 = 7

**性质5**：完全二叉树节点编号关系
- 节点i的父节点：⌊i/2⌋
- 节点i的左孩子：2i（若存在）
- 节点i的右孩子：2i+1（若存在）

### 任务3：特殊二叉树（15分钟）

**满二叉树**：
- 每层节点都达到最大值
- 深度k有 2^k - 1 个节点

**完全二叉树**：
- 除最后一层外，每层节点都达到最大值
- 最后一层节点集中在左边
- **用性质5计算编号关系**

**二叉排序树（BST）**：

**核心规则**：
- 左子树所有节点值 < 根节点值
- 右子树所有节点值 > 根节点值
- 左右子树本身也是BST（递归定义）

**图示示例**：
```
        50
       /  \
      30   70
     / \   / \
    20 40 60 80

验证：左子树都<50，右子树都>50 ✓
```

**查找过程**（查找60）：
```
        50        60 > 50，往右
       /  \
      30   70      60 < 70，往左
     / \   / \
    20 40 60 80    找到！

路径：50→70→60（3次比较）
```

**查找效率**：

| 情况 | 效率 | 原因 |
|------|------|------|
| 平均 | O(log n) | 树平衡，每次排除一半 |
| 最坏 | O(n) | 树退化成链表 |

**最坏情况**（插入序列已有序）：
```
    10
      \
       20
         \
          30      退化成链表，查找=O(n)
            \
             40
```

**考点**：
1. BST的中序遍历是升序
2. 最坏情况：插入序列已有序
3. 解决方案：用AVL、红黑树保证平衡

**平衡二叉树（AVL）**：

**核心特点**：每个节点的左右子树高度差不超过1

**平衡因子** = 左子树高度 - 右子树高度

| 平衡因子 | 状态 |
|---------|------|
| -1, 0, 1 | 平衡 ✓ |
| 其他值 | 不平衡，需旋转调整 |

**平衡的AVL示例**：
```
        50
       /  \
      30   70
     / \   /
    20 40 60

节点50：左高2，右高2，平衡因子=0 ✓
节点70：左高1，右高0，平衡因子=1 ✓
```

**不平衡示例**：
```
    50
   /
  30
 /
20

节点50：左高2，右高0，平衡因子=2 ❌ 需旋转！
```

**四种旋转操作**：

**旋转基本规则**：

**右旋（顺时针）**：左孩子上来，根下去变右孩子
```
    根              左孩子
   /      右旋      /    \
  左    ----->    左左    根
 / \                     /
左左 左右               左右

示例：
      50              30
     /      右旋      /  \
    30     ----->    20   50
   / \                    /
  20  40                40
```

**左旋（逆时针）**：右孩子上来，根下去变左孩子
```
  根                  右孩子
   \        左旋      /    \
    右      ----->   根     右右
   / \               \
  左右 右右            左右

示例：
    20                40
     \      左旋      /  \
      40   ----->   20   50
     / \              \
    30  50            30
```

| 旋转 | 方向 | 谁上来 | 谁下去 |
|------|------|--------|--------|
| 右旋 | 顺时针 | 左孩子 | 根变右 |
| 左旋 | 逆时针 | 右孩子 | 根变左 |

---

**1. LL型 - 右旋**（左子树的左孩子插入，左边过重）
```
插入前平衡：      插入后不平衡：        右旋后平衡：
      50               50                  30
     / \              / \                 /  \
    30  60    插入   30  60    右旋      20   50
   / \       --->   / \       --->      /    /  \
  20  40           20  40              10   40   60
                   /
                  10
```

**2. RR型 - 左旋**（右子树的右孩子插入，右边过重）
```
插入前平衡：      插入后不平衡：        左旋后平衡：
    20                20                   40
   / \               / \                  /  \
  10  40    插入    10  40     左旋      20   50
     / \    --->       / \    --->      / \    \
    30  50            30  50           10  30   60
                           \
                            60
```

**3. LR型 - 左右旋**（左子树的右孩子插入，先左旋再右旋）
```
初始：              插入40后不平衡：      左旋30后（变LL型）：   右旋50后：
    50                  50                    50                   40
   /                   /                     /                    /  \
  30        插入      30         左旋       40        右旋       30   50
            --->       \         --->      /         --->
                        40                 30

详细过程：
1. 插入40后：50的左高=2，右高=0，平衡因子=2，不平衡！
2. 左旋30：40上升到50的左边，30变成40的左孩子 → 变成LL型
3. 右旋50：40成为根，30在左，50在右 → 平衡！
```

**4. RL型 - 右左旋**（右子树的左孩子插入，先右旋再左旋）
```
插入后不平衡：      先右旋（60处）：     再左旋（20处）：
    20                  20                   40
     \                    \                 /  \
      60        右旋       40     左旋     20   60
     /         --->       / \    --->     / \    \
    40                  20  60           10  30   70
   / \                       \
  30  70                     70
```

**旋转总结**：

| 类型 | 插入位置 | 解决方法 | 记忆口诀 |
|------|---------|---------|---------|
| LL | 左孩子的左边 | 一次右旋 | 左左→右旋 |
| RR | 右孩子的右边 | 一次左旋 | 右右→左旋 |
| LR | 左孩子的右边 | 先左旋再右旋 | 左右→先左后右 |
| RL | 右孩子的左边 | 先右旋再左旋 | 右左→先右后左 |

**AVL vs BST 对比**：

| 特性 | BST | AVL |
|------|-----|-----|
| 查找效率 | 平均O(log n)，最坏O(n) | 稳定O(log n) |
| 插入效率 | O(log n)~O(n) | O(log n) |
| 空间开销 | 无 | 每节点存高度/平衡因子 |

**考点**：
1. 平衡因子 = 左高 - 右高，|平衡因子| ≤ 1
2. 四种旋转：LL右旋、RR左旋、LR左右旋、RL右左旋
3. 效率稳定：查找、插入、删除都是O(log n)

## 练习题目

1. 深度为4的满二叉树有多少个节点？
   - 答案：2^4 - 1 = 15

2. 某二叉树有100个节点，其中n2=30，问n0？
   - 答案：n0 = n2 + 1 = 31

3. 完全二叉树有100个节点，节点50的左孩子编号是？
   - 答案：2×50 = 100（正好是最后一个节点）

4. 完全二叉树有n个节点，深度是多少？
   - 答案：⌊log₂(n)⌋ + 1
