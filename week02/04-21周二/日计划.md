## 学习任务清单

### 任务1：顺序查找（15分钟）

**顺序查找原理**：
- 从第一个元素逐个比较
- 找到返回位置，否则返回失败

**性能分析**：

| 情况 | 时间复杂度 | 比较次数 |
|------|------------|----------|
| 最佳 | O(1) | 1次（第一个就找到） |
| 最坏 | O(n) | n次（最后一个或不存在） |
| 平均 | O(n) | n/2次 |

**适用场景**：
- 无序表
- 链表结构
- 小数据量

---

### 任务2：二分查找（45分钟）⭐重点

**二分查找原理**：
- 每次比较中间元素
- 根据比较结果缩小范围一半
- **前提：有序表**

**算法流程**：
```
BinarySearch(arr, key):
  low = 0, high = n-1
  while low <= high:
    mid = (low + high) / 2
    if arr[mid] == key:
      return mid
    elif arr[mid] < key:
      low = mid + 1
    else:
      high = mid - 1
  return -1  // 未找到
```

**性能分析**：

| 情况 | 时间复杂度 | 比较次数 |
|------|------------|----------|
| 最佳 | O(1) | 1次 |
| 最坏 | O(log n) | log₂n 次 |
| 平均 | O(log n) | log₂n 次 |

**二分查找次数计算**：
- n个元素最多比较 ⌈log₂(n+1)⌉ 次
- 例：100个元素最多比较 7 次（log₂101 ≈ 6.65 → 7）

---

## 练习题目

1. 有序数组[1,3,5,7,9,11,13]，查找7需要几次比较？
   - 答案：1次（中间位置直接找到）

2. 100个有序元素，二分查找最多几次比较？
   - 答案：⌈log₂101⌉ = 7次

3. 无序数组用什么查找？
   - 答案：顺序查找

4. 二分查找前提条件是？
   - 答案：有序表

---

## 易错点提醒

1. 二分查找必须有序
2. 计算比较次数用log₂(n+1)，不是log₂n
3. 二分查找仅适用于顺序存储（数组）
