## 学习任务清单

### 任务1：直接插入排序（20分钟）

**原理**：
- 每次将一个元素插入已排序序列
- 类似扑克牌整理

**算法流程**：
```
for i = 2 to n:
  key = arr[i]
  j = i - 1
  while j >= 0 and arr[j] > key:
    arr[j+1] = arr[j]
    j--
  arr[j+1] = key
```

**性能分析**：
- 时间复杂度：O(n²)
- 空间复杂度：O(1)
- 稳定性：**稳定**
- 适用：小数据、基本有序

---

### 任务2：冒泡排序（20分钟）

**原理**：
- 相邻元素比较交换
- 每趟最大元素"冒泡"到末尾

**算法流程**：
```
for i = 0 to n-1:
  for j = 0 to n-i-1:
    if arr[j] > arr[j+1]:
      swap(arr[j], arr[j+1])
```

**性能分析**：
- 时间复杂度：O(n²)
- 空间复杂度：O(1)
- 稳定性：**稳定**
- 特点：简单，效率低

---

### 任务3：简单选择排序（20分钟）

**原理**：
- 每趟选择最小元素放到已排序末尾

**算法流程**：
```
for i = 0 to n-1:
  minIdx = i
  for j = i+1 to n:
    if arr[j] < arr[minIdx]:
      minIdx = j
  swap(arr[i], arr[minIdx])
```

**性能分析**：
- 时间复杂度：O(n²)
- 空间复杂度：O(1)
- 稳定性：**不稳定** ⭐
- 特点：交换次数少

## 今日总结对比

| 算法 | 时间复杂度 | 空间复杂度 | 稳定性 | 特点 |
|------|------------|------------|--------|------|
| 直接插入 | O(n²) | O(1) | ✅稳定 | 基本有序时效率高 |
| 冒泡排序 | O(n²) | O(1) | ✅稳定 | 简单，效率低 |
| 简单选择 | O(n²) | O(1) | ❌不稳定 | 交换次数少 |

## 练习题目

1. 哪些排序算法是稳定的？
   - 答案：直接插入、冒泡

2. 简单选择排序为什么不稳定？
   - 答案：跨距离交换可能改变相等元素的相对位置

3. 基本有序的数据用哪种排序效率高？
   - 答案：直接插入排序
