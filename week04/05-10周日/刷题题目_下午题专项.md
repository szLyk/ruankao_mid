## 一、ER图补全解题技巧 ⭐重点

### 技巧1：找实体

**方法**：找题目描述中的名词

**示例**：
- "学生有学号、姓名" → 学生是实体
- "课程有课程号、课程名" → 课程是实体

---

### 技巧2：找属性

**方法**：找实体的特征描述

**示例**：
- "学生有学号、姓名、年龄" → 学号、姓名、年龄是属性

---

### 技巧3：找关系

**方法**：找实体间的联系

**示例**：
- "学生选课" → 选课是关系
- "学生属于班级" → 属于是关系

---

### 技巧4：判断联系类型

**方法**：看数量描述

| 描述 | 联系类型 |
|------|----------|
| "一个班级有多个学生" | 1:N |
| "多学生选多课程" | M:N |
| "一学生属于一班" | 1:N |

---

## 二、ER转关系模式解题技巧 ⭐重点必考

### 技巧：转换规则

| 联系类型 | 转换方式 |
|----------|----------|
| 每个实体 | 单独一个关系 |
| 1:1联系 | 合并到任一实体 |
| 1:N联系 | 合并到N端实体 |
| M:N联系 | 单独建立关系 |

---

### 示例：学生-课程-选课

**ER图**：
- 学生（学号、姓名）
- 课程（课程号、课程名）
- 选课（学生和课程的关系，有成绩属性）

**转换结果**：
```
学生关系：(学号, 姓名)
课程关系：(课程号, 课程名)
选课关系：(学号, 课程号, 成绩)  ← M:N单独建立
```

---

### 示例：学生-班级

**ER图**：
- 学生（学号、姓名）
- 班级（班级号、班级名）
- 属于（学生属于班级，1:N）

**转换结果**：
```
班级关系：(班级号, 班级名)
学生关系：(学号, 姓名, 班级号)  ← 1:N合并到N端
```

---

## 三、SQL语句填空解题技巧 ⭐重点

### 技巧：看前后语句逻辑

**常见填空位置**：
1. 表名
2. 列名
3. 条件（WHERE）
4. 连接条件（ON）
5. 聚合函数

---

### 示例

```sql
SELECT ___ FROM 学生 WHERE ___ = '张三'
```

**答案**：
- 第一个___：列名（如姓名、学号）
- 第二个___：列名（如姓名）

---

### 示例：JOIN填空

```sql
SELECT * FROM 学生 ___ 课程 ON 学生.学号 = ___
```

**答案**：
- 第一个___：JOIN
- 第二个___：选课.学号
